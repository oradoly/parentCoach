# PLANS.md

이 파일은 여러 파일 또는 계층을 변경하는 작업의 실행 계획을 기록한다. Codex는 복잡한 작업에서 코드를 작성하기 전에 이 양식을 채우고, 구현 중 발견된 사실에 따라 갱신한다.

## 현재 작업

- 상태: 미시작
- 작업명: 저장소 스캐폴딩 및 첫 수직 기능 계획
- 담당: Codex
- 관련 백로그: `docs/06_MVP_BACKLOG.md`의 M0

## 1. Goal

구체적으로 무엇을 만들거나 바꾸는가?

## 2. User outcome

이 작업이 부모 사용자의 어떤 문제를 해결하는가?

## 3. Context

반드시 읽은 파일, 관련 코드, 기존 제약을 적는다.

## 4. Product constraints

이 작업과 관련된 제품 불변조건과 비목표를 적는다.

## 5. Proposed changes

변경할 파일·모듈·API를 순서대로 적는다. 가능한 한 작은 수직 단위로 나눈다.

## 6. Data and API contract

새 입력·출력, 스키마 버전, 오류 형식, 마이그레이션 여부를 적는다.

## 7. Verification

- 단위 테스트:
- 통합 테스트:
- 수동 확인:
- 실행 명령:

## 8. Risks and mitigations

정확성, 개인정보, 비용, 지연, UX, 호환성 위험과 대응책을 적는다.

## 9. Done when

측정 가능한 완료 조건을 체크리스트로 적는다.

## 10. Progress log

작업 중 핵심 발견과 계획 변경을 날짜와 함께 기록한다.

## 11. Result

완료 후 변경 요약, 검증 결과, 남은 과제를 적는다.
