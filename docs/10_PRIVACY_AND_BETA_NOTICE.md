# 10. Privacy and Beta Notice Draft

이 문서는 내부 알파와 제한된 베타에서 사용자에게 안내할 개인정보·AI 이용 문구의 초안이다. 정식 개인정보 처리방침이 아니며, 공개 배포 전 국내 법률 검토가 필요하다.

## 1. 기본 원칙

- 부모코치는 초등학교 5~6학년 수학 문제 한 개를 부모가 설명하도록 돕는 도구다.
- MVP는 회원가입 없이 임시 세션으로 동작한다.
- 아이의 실명, 학교, 성적, 장기 학습 기록은 요구하지 않는다.
- 문제 사진과 문제 내용은 코칭 생성을 위해 필요한 기간만 처리한다.

## 2. 사진 촬영 안내

사진을 올리기 전에 다음을 권장한다.

- 문제 하나와 필요한 그림·표만 포함해 촬영한다.
- 이름, 학교명, 연락처, 얼굴, 다른 문제는 가능하면 제외한다.
- 여러 문제가 함께 보이면 하나만 잘라 올린다.
- 그림이나 표가 풀이 조건이면 잘리지 않게 포함한다.

## 3. 임시 처리와 삭제

- 원본 이미지는 임시 세션에만 사용한다.
- 세션은 짧은 TTL을 가지며, 사용자가 세션을 삭제하면 삭제 상태로 전환한다.
- 운영 로그에는 원본 이미지, 문제 전문, AI 프롬프트 전문, 원본 AI 응답 전문을 기본 저장하지 않는다.
- 장애 분석에 필요한 경우에도 요청 ID, 단계, 오류 코드, 지연 시간, 모델·스키마 버전처럼 원문 없이 볼 수 있는 정보만 우선 사용한다.

## 4. AI 결과 안내

- AI 인식 결과는 사용자가 확인·수정한 뒤에만 코칭 입력으로 사용한다.
- 최종 풀이와 비슷한 문제는 가능한 범위에서 별도 검산을 거친다.
- 검산이 실패하거나 모델을 일시 비활성화한 경우, 앱은 임의의 풀이를 보여 주지 않고 다시 시도하거나 문제를 확인하도록 안내한다.
- AI 결과는 부모의 설명 준비를 돕는 참고 자료이며, 아이의 능력이나 성향을 진단하지 않는다.

## 5. 운영 보호 장치

- 요청마다 진단용 request ID를 발급한다.
- request ID는 요청 단위 오류 추적용이며, 장기 사용자 식별자로 사용하지 않는다.
- 비용과 장애를 막기 위해 요청 제한이 적용될 수 있다.
- 요청이 잠시 제한되면 사용자는 잠시 뒤 다시 시도할 수 있다.
- 운영자가 모델 호출을 일시 중지할 수 있으며, 이 경우 앱은 안전 실패 문구를 보여 준다.

## 6. 공개 전 확인 필요

- 정식 개인정보 처리방침과 이용약관
- 외부 AI 처리 위탁 또는 국외 이전 고지 필요 여부
- 아동 개인정보 및 보호자 동의 관련 검토
- 저작권 있는 문제 이미지 처리 범위
- 오류 신고와 피드백 보관 기간
