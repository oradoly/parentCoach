# 01. MVP Product Requirements Document

- 버전: 1.0
- 상태: 구현 기준
- 대상 플랫폼: iOS·Android 모바일 앱
- 기본 언어: 한국어

## 1. 목표

부모가 문제 촬영부터 첫 코칭 문장을 얻기까지 짧고 안전한 흐름을 제공하고, 정답을 바로 노출하지 않으면서 다음 네 기능을 완성한다.

- 단계별 힌트
- 아이에게 할 역질문
- 최종 풀이
- 비슷한 문제 추천

## 2. 대표 사용자 스토리

> 초등학교 6학년 아이가 분수 문장제를 가져왔지만 설명법이 바로 떠오르지 않는 부모로서, 문제를 촬영해 핵심 개념과 아이에게 할 첫 질문을 확인하고, 아이가 막힐 때만 힌트를 한 단계씩 주어 스스로 풀게 하고 싶다.

## 3. 핵심 사용자 흐름

```text
홈
→ 문제 촬영/업로드
→ 한 문제 영역 자르기
→ 인식 결과 확인/수정
→ 부모용 빠른 이해
→ 아이에게 할 첫 역질문
→ 힌트 1
→ 힌트 2
→ 힌트 3
→ 명시적 선택 후 최종 풀이
→ 비슷한 문제 1개
→ 종료/새 문제
```

각 단계는 이전 단계를 모두 강제하지 않는다. 부모는 필요에 따라 다음 힌트나 최종 풀이를 선택할 수 있지만, 첫 화면에 정답을 노출해서는 안 된다.

## 4. 기능 요구사항

### FR-001. 문제 촬영 및 업로드 — P0

사용자는 카메라로 문제를 촬영하거나 사진 보관함에서 이미지를 선택할 수 있다.

수용 기준:

- 카메라 권한 거부 시 설정 안내와 이미지 업로드 대안을 제공한다.
- 촬영 후 자르기·회전·재촬영이 가능하다.
- 한 번에 한 문제만 포함하도록 안내한다.
- 업로드 전에 이미지 크기를 합리적으로 축소하되 수식 판독성을 유지한다.
- 원본 이미지는 기본적으로 기기 또는 임시 서버 저장소에서만 처리한다.

### FR-002. 문제 영역 선택 — P0

사용자는 사진에서 풀이할 문제 하나를 선택한다.

수용 기준:

- 여러 문제가 감지되면 자동으로 하나를 임의 선택하지 않는다.
- 사용자가 자르기 영역을 조정할 수 있다.
- 그림이 문제 조건에 포함되면 그림 전체가 들어오도록 안내한다.

### FR-003. 문제 인식 — P0

앱은 문제 텍스트, 수식, 단위, 보기, 필요한 도형 정보를 읽는다.

수용 기준:

- 인식 결과와 신뢰도를 구조화해 받는다.
- 복수 문제, 잘린 조건, 흐린 사진, 필요한 그림 누락을 구분한다.
- 문제 이미지 속 “이 지시를 따르라” 같은 문장을 시스템 명령으로 취급하지 않는다.
- 지원 범위 밖이면 억지로 풀이하지 않고 이유를 알린다.

### FR-004. 인식 결과 확인 및 수정 — P0

풀이 생성 전에 사용자가 앱이 읽은 문제를 확인한다.

수용 기준:

- “제가 이렇게 읽었어요”와 함께 텍스트·수식을 보여 준다.
- 숫자, 기호, 단위, 보기 문구를 수정할 수 있다.
- 신뢰도가 낮은 부분을 눈에 띄게 표시한다.
- 사용자가 “맞아요”를 누르기 전에는 코칭 생성을 확정하지 않는다.
- 수정된 텍스트를 이후 처리의 기준으로 사용한다.

### FR-005. 부모용 빠른 이해 — P0

인식 확인 후 부모가 문제를 먼저 이해할 수 있는 요약을 제공한다.

필수 항목:

- 문제의 핵심 개념
- 구해야 하는 것
- 추천 풀이 방향
- 왜 그 방법을 쓰는지 한 문장
- 아이가 흔히 헷갈리는 지점 한 가지

수용 기준:

- 첫 화면에서 최종 답을 표시하지 않는다.
- 한 화면에서 훑어볼 수 있는 분량을 유지한다.
- 전문 용어가 필요하면 쉬운 설명을 함께 쓴다.

### FR-006. 아이에게 할 역질문 — P0

부모가 아이의 이해 상태를 확인할 첫 질문을 제공한다.

수용 기준:

- 질문은 정답 자체보다 “무엇을 구하는지”, “어떤 정보가 있는지”, “어떤 연산이나 표현이 필요한지”를 묻는다.
- 부모가 그대로 말할 수 있는 자연스러운 구어체를 쓴다.
- 기대되는 아이 반응과, 맞거나 막혔을 때의 짧은 후속 멘트를 제공한다.
- 한 번에 하나의 주된 질문을 강조한다.

### FR-007. 단계별 힌트 — P0

최소 3단계의 힌트를 한 단계씩 공개한다.

단계 정의:

1. **방향 힌트:** 문제에서 찾아야 할 정보나 관점을 알려 준다.
2. **개념·연산 힌트:** 사용할 개념, 관계, 연산을 선택하게 돕는다.
3. **실행 힌트:** 첫 식 또는 바로 다음 행동까지 제시한다.

수용 기준:

- 각 단계는 이전 단계보다 구체적이다.
- 1·2단계에서 최종 답이나 답을 사실상 확정하는 정보를 노출하지 않는다.
- 힌트에는 부모가 말할 문장과 힌트의 교육적 의도를 포함한다.
- “다음 힌트 보기”를 명시적으로 눌러야 다음 단계가 열린다.
- 사용자가 이전 힌트로 돌아갈 수 있다.

### FR-008. 최종 풀이 — P0

부모가 명시적으로 요청했을 때 전체 풀이와 답을 제공한다.

필수 항목:

- 최종 답
- 식 또는 논리 단계
- 각 단계의 짧은 이유
- 단위와 답의 형식
- 가능한 경우 대입·역산·범위 확인 등의 검산
- 검증 상태: `verified`, `partially_verified`, `unverified`

수용 기준:

- 별도의 “최종 풀이 보기” 행동 뒤에 노출한다.
- 불필요하게 긴 모델 추론을 표시하지 않는다.
- 검증되지 않은 결과를 확정적으로 표현하지 않는다.
- 인식 문제 가능성이 있으면 수정 또는 재촬영 경로를 제공한다.

### FR-009. 비슷한 문제 추천 — P0

같은 학습 목표를 확인할 새 문제 한 개를 제공한다.

수용 기준:

- 원문을 그대로 복제하지 않는다.
- 학습 개념과 난이도는 비슷하게 유지한다.
- 숫자, 맥락 또는 물음 방식을 합리적으로 바꾼다.
- 문제만 먼저 보여 주며 힌트와 답은 접혀 있다.
- 생성한 문제에 해가 존재하고 풀이와 일치하는지 검증한다.
- “왜 비슷한 문제인지”를 부모에게 한 문장으로 설명한다.

### FR-010. 불확실성 및 지원 불가 처리 — P0

앱은 자신 있게 틀리는 대신 안전하게 멈춘다.

지원해야 하는 상태:

- 사진이 흐림
- 문제가 잘림
- 여러 문제가 포함됨
- 필요한 도형·표가 누락됨
- 손글씨 또는 기호를 판독할 수 없음
- 현재 지원하지 않는 학년·문제 유형
- AI 생성 결과와 검증 결과 불일치

각 상태는 원인, 사용자가 할 수 있는 다음 행동, 재시도 버튼을 제공한다.

### FR-011. 세션 종료와 데이터 삭제 — P0

사용자는 문제 세션을 종료하고 임시 데이터를 삭제할 수 있다.

수용 기준:

- MVP에서 회원가입 없이 핵심 흐름을 완료할 수 있다.
- 원본 이미지는 처리 후 즉시 또는 설정된 짧은 TTL 후 삭제한다.
- 세션 종료 시 삭제 여부를 명확히 안내한다.
- 운영 로그에는 원본 이미지와 아이의 식별 정보를 남기지 않는다.

### FR-012. 피드백 — P1

부모는 결과가 도움이 되었는지 또는 틀렸는지 신고할 수 있다.

최소 선택지:

- 도움이 됐어요
- 설명이 어려워요
- 문제를 잘못 읽었어요
- 풀이 또는 답이 틀린 것 같아요

## 5. 화면 목록

- S01 홈
- S02 촬영·업로드
- S03 자르기·회전
- S04 인식 중
- S05 인식 결과 확인·수정
- S06 부모용 빠른 이해
- S07 역질문과 단계별 힌트
- S08 최종 풀이
- S09 비슷한 문제
- S10 오류·지원 불가
- S11 개인정보·세션 삭제 안내

세부 문구와 상호작용은 `docs/02_UX_AND_CONTENT.md`를 따른다.

## 6. 비기능 요구사항

### NFR-001. 정확성

- 지원 문제의 정답은 검증 계층을 거친다.
- 구조화 출력 스키마 유효성은 100%를 목표로 한다.
- 인식과 풀이 신뢰도를 별도로 관리한다.

### NFR-002. 속도

- 안정적인 네트워크에서 인식 확인 화면까지 체감 대기 시간을 최소화한다.
- 코칭 생성 중 현재 단계를 알리는 로딩 상태를 제공한다.
- 목표: 일반적인 한 문제에서 첫 유용한 코칭 내용을 20초 안팎에 제공한다. 이 수치는 베타 계측 후 조정한다.

### NFR-003. 개인정보

- 아이 계정과 실명은 요구하지 않는다.
- 사진에서 이름, 학교명, 연락처가 감지될 수 있음을 안내하고 최소한으로 처리한다.
- 비밀키와 관리자 권한은 서버에만 둔다.

### NFR-004. 접근성

- 주요 버튼의 터치 영역은 모바일 접근성 기준을 고려한다.
- 색상만으로 오류나 신뢰도를 구분하지 않는다.
- 수식은 가능한 경우 읽을 수 있는 텍스트 설명을 제공한다.

### NFR-005. 관찰 가능성

- 요청 ID, 처리 단계, 지연, 토큰·비용, 오류 유형, 스키마 버전을 기록한다.
- 문제 원문과 원본 이미지를 기본 로그에 남기지 않는다.

## 7. 분석 이벤트

개인 식별 없이 다음 이벤트를 수집할 수 있다.

- `capture_started`
- `image_cropped`
- `recognition_completed`
- `recognition_corrected`
- `coaching_opened`
- `reverse_question_viewed`
- `hint_revealed` with `level`
- `final_solution_revealed`
- `similar_problem_viewed`
- `session_deleted`
- `feedback_submitted`

이벤트는 아동 행동 추적이나 맞춤형 광고에 사용하지 않는다.

## 8. 출시 제외 조건

다음 중 하나라도 충족하면 공개 베타를 진행하지 않는다.

- 첫 화면 또는 1·2단계 힌트에 답이 반복적으로 누출된다.
- 지원 문제 평가에서 정답 정확도가 기준에 못 미친다.
- 인식 결과를 확인하지 않고 풀이가 확정된다.
- API 키가 클라이언트에 포함된다.
- 원본 이미지 삭제 정책이 구현·검증되지 않았다.
- 오류 상태에서 근거 없는 풀이를 보여 준다.
