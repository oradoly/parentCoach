# 14. Internal Alpha Observation Template

이 문서는 내부 알파 1회 테스트를 기록하는 템플릿이다. 문제 전문, 원본 이미지, 아이 식별 정보는 기록하지 않는다.

## 1. 작성 원칙

- 테스트 1회에 문서 1개 또는 표 1행만 작성한다.
- 참여자와 아이를 직접 식별할 수 있는 정보는 쓰지 않는다.
- 문제는 `domain`과 `skill`로만 분류한다.
- 관찰 메모에는 UX, 문구, 오류 원인만 남긴다.
- requestId는 오류 추적용으로만 기록하고 장기 사용자 식별자로 쓰지 않는다.

## 3. P0 중단 조건

아래 항목은 발견 즉시 파일럿을 멈추고 수정 대상으로 분류한다.

- 인식 확인 전 코칭이 생성된다.
- 첫 코칭 화면 또는 힌트 1·2에 최종 답이 노출된다.
- 검산 실패, 모델 비활성화, 지원 불가 상태에서 임의 풀이가 노출된다.
- 원본 이미지, 문제 전문, 아이 식별 정보가 기록 또는 로그에 남는다.
- 부모가 앱을 아이에게 직접 답 확인 도구로 넘겨 쓰게 된다.
- 참여자가 중단을 요청한다.

## 4. P1 개선 후보

아래 항목은 파일럿을 계속할 수 있지만 다음 개선 목록에 넣는다.

- 촬영 안내가 부족해 재촬영이 반복된다.
- 인식 수정 화면의 문구가 헷갈린다.
- 부모가 첫 역질문을 그대로 말하기 어렵다.
- 힌트 단계 차이가 작거나 너무 갑자기 어려워진다.
- 최종 풀이가 부모에게 길거나 딱딱하게 느껴진다.
- 비슷한 문제가 원문과 너무 비슷하거나 난이도가 맞지 않는다.
- 피드백 선택지가 실제 느낌과 맞지 않는다.

## 5. 집계 표 권장 열

스프레드시트로 모을 때는 아래 열만 사용한다.

```text
test_id
date
grade_band
domain
skill
entry_path
time_to_opening_question_seconds
recognition_edit_type
recognition_failure_type
opening_question_sayable
blocked_step
hints_opened
final_solution_opened
similar_problem_opened
similar_solution_opened
feedback_choice
error_stage
request_id
p0_stop
p0_reason
p1_candidate
next_action
```

집계 표에도 문제 전문, 이미지 링크, 참여자 실명, 아이 이름, 학교명은 넣지 않는다.
